Router

Используйте Rox Router, чтобы получить динамическое управление путями.
К примеру у вас есть кнопка "Route Me", которая должна вас направить на какой-то URL адрес (в рамках Origin), но переходит к URL адресу должен выполниться бесшовно и незаметно для пользователя, страница не должна перегружаться, а обновиться должен конкретный блок/элемент.

В этом поможет Rox Router, он умеет бесшовно менять контент страницы как выборочно, так и полностью. А так же поддерживает плавную смену контента.
Использование (Front / JS)
Объяснение кода
Мы получаем кнопку button и добавляем ей отслеживание события click, которое произойдет, если по кнопке кликнуть. Далее выполняется функция Router.go(), которая принимает 4 аргумента:
  • URL - адрес страницы. Если мы перемещаемся уровнями выше, тогда указывать / не нужно. Если мы находимся по адресу /audio/123 и нам нужно перейти по адресу ниже, чем /audio/, тогда мы используем слэш (/) и адрес, куда нам нужно перейти.
  • Selector - элемент на странице, который нам нужно обновить. Не обязательный параметр, в случае отсутствия его и последующих других, будет обновлена полностью страница.
  • getSelector - если мы знаем селектор элемента запрашиваемой страницы, мы можем только его информацию запросить и обновить в Selector (не обязательно)
  • AnimationID - принимает ID анимации, которая будет проигрываться между переходами страниц или обновлением контента.
Данный роутинг используется практически во всех SPA приложениях, где используется Server-Side Rendering. Но что делать, если у нас есть только JS-компоненты с HTML структурой, которые динамически отображают контент по запросу, без физических путей и SSR?

Virtual Router

Rox Virtual Router - это концепция абстракции физических путей в виртуальном пространстве, которое получает данные из единого экземпляра и динамически инициализирует HTML-код. Этот подход отлично подходит для разработки PWA.

Виртуальный роутинг достигается за счет использования синхронизации вкладок в BroadcastChannel и данных в sessionStorage. Рендеринг страницы выполняется после подгрузки динамических компонентов.
Использование (Front / JS)
Объяснение кода
В этот раз мы динамически импортируем необходимые пользовательские компоненты, в которых по умолчанию используется функция депортирования HTML-кода. Как вы могли заметить, первые два импорта напрямую добавляют JS-файлы, но последний Content не определен расширением JS. Что происходит дальше?

Content импортируется непосредственно как компонент, который включает в себя: index.js, media.js, functions.js, deport_module.js

Что позволяет импортировать полноценный компонент и работать с его функциями. По умолчанию должен депортироваться HTML-код, так как он используется для рендеринга.

Следующий код предполагает переход в виртуальный путь /music. Так и происходит, но эта же функция является генератором и работает как middleware. Вы можете контролировать переход по путям и с помощью функции next() выполнять их.
Параметры Router.go(...)
  • PATH - адрес виртуального пути. Все пути должны быть зарегистрированы и переданы Virtual Router при его инициализации или динамически дополнены в будущем.
  • Object (click, target) - в объекте click должен быть передан элемент, на который вешается триггер клика, а в объекте target может содержаться элемент страницы, который в своем теле отобразит контент. По умолчанию, если target отсутсвует, то переход применяется ко всей странице
  • next - функция генератор, которая выполняет рендеринг и переход по пути. В случае, если next() отсутствует переход к PATH будет заблокирован.
